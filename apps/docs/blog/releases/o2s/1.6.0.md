We're excited to announce Release 1.6.0 of Open Self Service! This release delivers a complete **cart and checkout flow** with Medusa integration, a powerful **interactive CLI wizard** for project bootstrapping, a major **domain-based restructuring** of blocks and UI components, and expanded **Zendesk Help Center** support for articles and categories.

Let's dive into what's new!

<!--truncate-->

## New features

### Cart and checkout flow

This release introduces end-to-end commerce capabilities with a complete cart and checkout implementation:

- **New blocks**: Cart, Checkout Summary, Checkout Shipping Address, Checkout Company Data, Checkout Billing & Payment, and Order Confirmation — each with full API harmonization, frontend, and SDK layers.
- **Cart management**: full lifecycle support including adding/removing items, promo codes, and persistent cart ID via `localStorage`. "Add to Cart" buttons are now available across Product List, Product Details, and Recommended Products blocks.
- **Checkout forms**: multi-step checkout with validation (Formik + Yup), address fields, company data, and payment session handling.
- **Medusa integration**: Carts (CRUD, items, promotions, shipping), Checkout (addresses, shipping, payment, place/complete order), Customers (address CRUD with set-default), and Payments (provider listing, session management) — all fully integrated with the Medusa.js backend.
- **New UI components**: `StepIndicator`, `RadioTile`, `AddressFields`, `CartSummary`, `CartItem`, `CartPromoCode`, and `QuantityInput`.
- **Navigation**: cart icon added to header (desktop + mobile) with CMS-driven URL and label.

:::tip
For detailed setup, data models, and integration guides, see the new documentation for [Carts](../../../docs/main-components/harmonization-app/normalized-data-model/carts), [Checkout](../../../docs/main-components/harmonization-app/normalized-data-model/checkout), and [Payments](../../../docs/main-components/harmonization-app/normalized-data-model/payments).
:::

### Interactive CLI wizard (`create-o2s-app`)

We introduced a completely new interactive CLI wizard for bootstrapping O2S projects, similar to `create-next-app`:

- **Three template paths**: O2S (Self Service Portal), DXP (Digital Experience Platform), and Custom (pick individual blocks & integrations).
- **Dynamic package discovery**: reads `o2sTemplate` metadata from each package's `package.json` to categorize blocks and integrations per template.
- **Guided setup**: template selection → block selection → integration selection → environment variable prompts → scaffolding with automatic config transformation.
- **Branding and UX polish**: branded CLI banner, boxed summary with next steps, docs links, and actionable warnings after project creation.
- **New `@o2s/integrations.mocked-dxp`** integration providing CMS, articles, search, and mock content specifically for the DXP template path.

```shell
npx create-o2s-app
```

### Domain-based blocks and UI restructuring

We introduced a structural refactor that reorganizes blocks and UI components around business domains:

- **Blocks** are now grouped under `packages/blocks/<domain>/<block>` (e.g., `billing/invoice-list`, `support/ticket-details`, `checkout/cart`, `content/hero-section`).
- **UI components** are reorganized into domain groups: `layout`, `data`, `feedback`, `cards`, `charts`, `commerce`, `content`, `navigation`, `forms`, `actions`, `media`.
- Updated all dependent imports across frontend, blocks, templates, and Storybook.
- Defensive rendering now returns `null` for incomplete block payloads instead of crashing at runtime.

:::info
**Breaking change**: old UI import paths no longer work. The new format is `@o2s/ui/components/<group>/<Component>`.
:::

### Common block prop types

Block prop definitions have been standardized using shared framework models:

- New shared types: `BaseBlockProps`, `BlockWithSlugProps`, `BlockWithUserIdProps`, `BlockWithDraftModeProps`, `FullBlockProps`.
- Removed duplicated local prop definitions (`slug`, `userId`, `isDraftModeEnabled`) across all blocks.
- Aligned `RendererProps` contracts for consistency and easier maintenance.

### Zendesk Help Center integration

The Zendesk integration now supports Help Center article and category management:

- Integrated Zendesk Help Center API with dedicated articles module (service, mapper, controller).
- Added Help Center OpenAPI specification for type-safe API access.
- Category fetching now handles missing items gracefully, search reports accurate totals.
- Mocked pages and content for Zendesk-based article flows.

:::tip
See the updated [Zendesk integration documentation](../../../docs/integrations/articles/zendesk/overview) for setup instructions and available features.
:::

### Page-level redirects

Added support for CMS-driven page redirects:

- New optional `redirect` field in the CMS Page model (`@o2s/framework`).
- Frontend performs locale-aware Next.js redirect when `meta.redirect` is present.
- Configurable per page through CMS — useful for landing pages, locale-specific homepages, and URL migrations.

## Other changes

### Developer experience improvements

This release includes several improvements to the development workflow and code quality:

- **Service inheritance alignment**: integration services now extend base service classes for proper NestJS DI compatibility.
- **Typed header constants**: replaced "magic string" headers with shared, typed constants across framework and api-harmonization.
- **Block registry**: the frontend `renderBlocks` switch was replaced with a typed block registry, making it easier to extend and diagnose missing block types.
- **Improved generators**: better CMS integration scaffolding, cleaner block generator output, and corrected `eject-block` source branch handling.

### Storybook and documentation

- Custom Storybook landing page with O2S branding, welcome banner, and navigation guidance.
- Package-level README.md files are now used as Storybook documentation (autodocs enabled across blocks).
- Updated READMEs across blocks, integrations, and framework packages.
- New comprehensive documentation for Carts, Checkout, Customers, Orders, Payments, and Resources data models.

### Testing and quality

- **Integration unit tests**: extensive new test coverage for Algolia, Zendesk, Redis, MedusaJS, Contentful, and Strapi — covering services, mappers, error handling, and SDK delegation.
- **Vitest infrastructure**: standardized configuration with coverage collection, merging, and reporting automation.
- Fixed carousel keyboard handling to prevent multiple carousels from responding simultaneously.
- Fixed duplicated navigation items rendering.
- Various defensive rendering and error-handling improvements across blocks.

### Dependency and security updates

As always, we're committed to maintaining a stable and secure platform. This release includes various bug fixes and security updates, including:

- **Storybook 10.2** across the stack.
- Broad dependency refresh across framework, integrations, CLI, and UI packages.
- Security patches for vulnerable dependencies (axios, flatted, qs, swiper).
